Configuration Guide

Quick fix: Run the AMI setup script on your node:
sudo bash /opt/ASL3-EZ/ami-setup.sh
This reads manager.conf, tests the connection, and updates the service file automatically.

Step 1 — /etc/asterisk/manager.conf must have a user stanza:

[general]
enabled = yes
port = 5038
bindaddr = 127.0.0.1

[admin]
secret = your_secret_here
read = system,call,log,verbose,command,agent,user,config,dtmf,reporting,cdr,dialplan
write = system,call,log,verbose,command,agent,user,config,dtmf,reporting,cdr,dialplan
permit = 127.0.0.1/255.255.255.0

Step 2 — Set credentials in the service file:

sudo nano /etc/systemd/system/ASL3-EZ.service
# Set these two Environment lines:
Environment="AMI_USER=admin"
Environment="AMI_SECRET=your_secret_here"

Step 3 — Apply changes:

sudo systemctl daemon-reload
sudo systemctl restart ASL3-EZ
sudo asterisk -rx "module reload manager"

Flask Secret Key

SECRET_KEY is a Flask app setting. It ships with a generic default so the app works out of the box; changing it is good practice, especially once login/session features are added. It's stored as an Environment line in the systemd unit file.
