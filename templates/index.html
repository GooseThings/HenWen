Quick Guide — DTMF Macros

►

Each entry maps a numeric slot to a DTMF command sequence. The slot number is how the macro is referenced — by a user dialing *3<slot> on the radio, by a schedule entry, or by another macro.

Allowed characters in a command sequence:

Character(s)Meaning
0–9 * # A–DDTMF digits sent to app_rpt
p~500 ms pause (lowercase or uppercase P)
(space)Separates multiple commands in one slot

Common app_rpt commands you can put in a macro:

CommandWhat it does
*3<node>Connect to a node (transceive)
*1<node>Disconnect a specific node
*70Disconnect all linked nodes
*80System status announcement
*99Restart app_rpt (reload rpt.conf)

Example entries:

1 = *327096       ; slot 1: connect to node 27096
2 = *170 p *327096 ; slot 2: disconnect all, pause, connect 27096
3 = *70            ; slot 3: disconnect all nodes

The node stanza must have macro = {{ m }} (or the default stanza named macro) for these entries to be active.

Quick Guide — Scheduled Macros

►

Each entry runs a macro slot at a given time. The key is the macro slot number (from the node's macro stanza), and the value is a five-field cron-style time expression.

Time field order: minute   hour   day-of-month   month   day-of-week

FieldRangeExample
minute0–5930 = at :30
hour0–2306 = 6 AM
day of month1–31* = every day
month1–12* = every month
day of week0–6 (Sun=0)1 = Monday

Example entries:

1 = 00 06 * * *   ; run macro slot 1 every day at 06:00
2 = 30 18 * * 5   ; run macro slot 2 every Friday at 18:30
3 = 00 12 1 * *   ; run macro slot 3 on the 1st of each month at noon

Only plain numbers and * are supported per field — ranges (1-5), lists (1,3,5), and steps (*/2) are not supported by app_rpt and will be rejected on save. The node stanza must have scheduler = {{ s }} (or the default stanza named schedule) for entries here to fire.

Configuration Guide

Quick fix: Run the AMI setup script on your node:
sudo bash /opt/ASL3-EZ/ami-setup.sh
This reads manager.conf, tests the connection, and updates the service file automatically.

Step 1 — /etc/asterisk/manager.conf must have a user stanza:

[general]
enabled = yes
port = 5038
bindaddr = 127.0.0.1

[admin]
secret = your_secret_here
read = system,call,log,verbose,command,agent,user,config,dtmf,reporting,cdr,dialplan
write = system,call,log,verbose,command,agent,user,config,dtmf,reporting,cdr,dialplan
permit = 127.0.0.1/255.255.255.0

Step 2 — Set credentials in the service file:

sudo nano /etc/systemd/system/ASL3-EZ.service
# Set these two Environment lines:
Environment="AMI_USER=admin"
Environment="AMI_SECRET=your_secret_here"

Step 3 — Apply changes:

sudo systemctl daemon-reload
sudo systemctl restart ASL3-EZ
sudo asterisk -rx "module reload manager"

About ASL3-EZ

A browser-based manager for AllStarLink 3 nodes — rpt.conf editing with live validation, node connect/disconnect via AMI, dashboard, backups, and more.
